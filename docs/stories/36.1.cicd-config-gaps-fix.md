# Story 36.1 — Fix CI/CD Configuration Gaps

**Priority:** High
**Effort:** Small (YAML config only)
**Type:** Bug / Hardening
**Status:** Done

---

## Context

During a CI/CD review, 6 configuration gaps were identified in `.github/workflows/ci.yml` and `deploy.yml`. The most critical: G3 (Security) scan failures were not blocking deploys to staging/production.

---

## Acceptance Criteria

### AC 36.1.1 — G3 Security gate blocks deploy
- **Given** the security scan (G3) fails
- **When** the deploy workflow runs
- **Then** staging deploy is blocked until G3 passes

### AC 36.1.2 — Console rollback in G6
- **Given** smoke tests fail in G6
- **When** the rollback step runs
- **Then** both Railway services AND Vercel Console are rolled back to the previous deployment

### AC 36.1.3 — Expanded smoke tests
- **Given** staging is deployed
- **When** G6 smoke tests run
- **Then** at minimum these are verified:
  - `/health` is reachable
  - Auth guard returns 401/403 on at least 2 protected endpoints
  - Console URL returns 2xx/3xx
  - Unknown routes return 404 (not 500)

### AC 36.1.4 — PNPM version consistency
- **Given** `package.json` declares `packageManager: pnpm@10.32.1`
- **When** CI/CD workflows install pnpm
- **Then** version is inferred from `packageManager` field (no hardcoded version in workflows)

### AC 36.1.5 — Test secrets use fallback pattern
- **Given** repo secrets `TEST_ENCRYPTION_SECRET`, `TEST_JWT_SECRET`, `TEST_SUPABASE_JWT_SECRET` may or may not be configured
- **When** G2 integration tests run
- **Then** secrets are sourced from repo secrets with inline fallback defaults

### AC 36.1.6 — Rollback strategy improved
- **Given** smoke tests fail
- **When** Railway rollback runs
- **Then** rollback uses `git checkout $SHA -- .` (file checkout) instead of `git checkout $SHA` (detached HEAD)

## Changes Made

| File | Change |
|---|---|
| `.github/workflows/deploy.yml` | Added G3 wait-on-check in `ci-check` job |
| `.github/workflows/deploy.yml` | Added "Rollback Console on Failure" step using `vercel promote` |
| `.github/workflows/deploy.yml` | Expanded smoke tests from 2 to 5 checks |
| `.github/workflows/deploy.yml` | Removed hardcoded pnpm version from `pnpm/action-setup` |
| `.github/workflows/deploy.yml` | Fixed git checkout in rollback to use `-- .` |
| `.github/workflows/ci.yml` | Removed `PNPM_VERSION` env var, pnpm/action-setup reads `packageManager` |
| `.github/workflows/ci.yml` | Secrets now use `${{ secrets.X \|\| 'fallback' }}` pattern |

## Risks

- **Vercel rollback** depends on `vercel list --meta gitSha` finding the previous deployment. If the previous commit was never deployed to Vercel, rollback will log a warning but not fail the workflow.
- **G3 adding latency** to deploy: CodeQL can take 3-5 min. Deploy now waits for it. This is intentional (security > speed).
